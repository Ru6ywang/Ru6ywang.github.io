3.5 控制访问

3.5.1 包的概念

​ 1、把功能相似或相关的类或接口组织在一个包中，方便查找和使用。

​ 2、包采用了树形目录的存储形式。同一个包中的类名字是不同的，不同的包中类名可以是相同的。当同时调用两个不同包中相同类名的类时，应该加上包名来区分。因此，包可以避免名字冲突。

​ 3、包也限定了访问权限，只有拥有包访问权限的类，才能访问某个包中的类。

在实际开发中，包结合import将关键字和权限修饰一同使用，可达到控制访问的目的。

​ 4、在程序中给包名是，通常是将域名倒着来写。例如www.abc.com 则 包名是 com.abc

​ 5、在使用package 关键字新建包并在包中定义类后，如果再使用这个类则需要使用import关键字导入。使用包应结合权限修饰来对文件进行管理。

​ 例：package three.com; //新建包

​ import com.abc.Student;//引入包

3.5.2 权限修饰符

​ 1、权限修饰符包含四种：private、default、protected和public。

​ 2、public 修饰内外部类、成员变量、成员方法，对项目中的类是可见的，但在不同的包中访问是需要导入包的。

​ 3、default 修饰内外部类、成员变量、成员方法，对项目中同一个包的类是可以见的。

​ 4、protected 修饰内部类、成员变量、成员方法，对项目中同一个包中的类可见或子类可见。

​ 5、private 修饰内部类、成员变量、成员方法，对项目内除自身外其他类都不可见。

3.5.3. 内部类

  • 内部类：在一个类的内部定义一个类。内部类作为外部类的成员，依附于外部类存在。

  • 内部类可以是静态的，也可以用四种权限修饰符进行修饰。

  • 内部类有四种形式：成员内部类、局部内部类、静态内部类和匿名内部类。

    1、 成员内部类：与成员变量一样，属于类的成员变量。

      ………………………

      }

      ………………………

      }

    • 如果使用内部类就需要一个外部类实例化一个对象。

      • 例：outer o =new Outer();

        Outer.Inner oi = o.new Inner();

    1、 只有实例化对象后才能实例化成员内部类的成员变量和成员方法。

    2、 内部类的权限修饰不能大于外部类的权限修饰，否则程序会出错。

    3、 内部类可以使用与外部类相同的成员变量或成员方法名，某种意义上内部类与外部类的功能是一样的，只不过内部类在外部类的类体中。

  • 局部内部类声明在成员方法体内，作用范围只能是该方法体。

局部内部类类似方法的局部变量，所以在类外或类的其他方法中不能访问这个内部类。这并不代表局部内部类的实例和定义了他的方法中的局部变量具有相同的生命周期。

只能在方法内部类（局部内部类）定义后使用，不存在外部可见性问题，因此没有访问修饰符。不能在局部内部类中使用可变的局部变量，可以访问外围类的成员变量。如果static方法，则只能访问static修饰的成员变量，可以使用final或者abstract 修饰。

静态内部类

1、静态内部类和静态成员变量类似，都是使用static修饰的。

在创建静态内部类时，不需要将静态成员的实例绑定在外部类的实例上。普通非静态内部类的对象是依附在外部类对象之中的。要在一个外部类中定义一个静态的内部类，不需要利用关键词new来创建内部类的实例。静态类和方法只属于类的本身，并不属于该类的对象，更不属于其他外部类的对象。

2、要创建静态内部类的对象，并不需要其他外部类的对象，不能从静态内部类的对象中访问非静态的外部类对象。

使用匿名内部类的规则如下：

（1）使用匿名内部类时，必须是继承一个类或者实现一个接口，但是两者不可兼得，同时也只能继承一个类或实现一个接口。

（2）匿名内部类中是不能定义构造函数的。

（3）匿名内部类中不能存在任何的静态成员变量和静态方法。

（4）匿名内部类为局部内部类，所以局部内部类的所有限制同样对匿名内部类生效。

（5）匿名内部类不能是抽象的，它必须要实现继承的类或实现接口的所有抽象方法。
